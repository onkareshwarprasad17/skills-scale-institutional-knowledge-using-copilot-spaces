# OctoAcme Project Management Docs

This folder contains OctoAcme’s project management process documents and quick guidance to help new team members find the right artifacts and follow our cadence.

OctoAcme runs projects iteratively with clear ownership and measurable outcomes. Projects move through Initiation (one‑pager, stakeholder alignment, go/no‑go), Planning (kickoff, prioritized backlog, Definition of Done, release plan), Execution (timeboxed sprints, PR workflow, automated CI), Release (pre‑release checks, staging smoke tests, rollback plan), and Close/Retrospective (action items and follow up). Key artifacts include the Project One‑pager, roadmap and release plan, sprint backlog, Acceptance Criteria & DoD, Risk Register, and retrospective action items.

Team roles and communication are explicit: Product Managers define outcomes and metrics; Project Managers coordinate schedules, risks, and stakeholder communications; Developers build and test with an emphasis on testability; QA validates acceptance criteria; stakeholders provide approvals. Regular cadences (daily standups, weekly delivery syncs, sprint demos, monthly stakeholder updates) plus templates (weekly status, release notes, incident summaries) keep everyone aligned.

Quality and risk practices are integrated into each stage. Planning includes an initial test approach and DoD; CI enforces automated tests, linting, and security scans; manual QA and smoke tests are used as needed before production. Risks are tracked in a Risk Register and reviewed during weekly syncs; incidents follow an escalation and blameless retrospective process.
